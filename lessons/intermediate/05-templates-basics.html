模板基础

函数模板 类模板 参数推导 模板特化 非类型参数

🎯 学习目标

  • ✓ 理解模板的作用：用一套代码适配多种类型
  • ✓ 掌握函数模板的定义与调用方式
  • ✓ 掌握类模板的定义，实现泛型容器（如 Stack<T>）
  • ✓ 理解编译器的模板参数推导规则
  • ✓ 了解全特化与非类型模板参数的用法

⚙️ 1. 函数模板

模板让我们用一份代码处理多种类型，避免为每个类型重复编写相同逻辑。函数模板以 template<typename T> 开头，T 是占位类型，由编译器在调用时自动推导或显式指定。

基本函数模板 
#include <iostream>
#include <string>

// 函数模板：返回两个值中的较大者
template<typename T>
T myMax(T a, T b) {
    return (a > b) ? a : b;
}

// 多类型参数模板
template<typename T, typename U>
void printPair(T first, U second) {
    std::cout << "(" << first << ", " << second << ")\n";
}

// 模板函数：交换两个变量
template<typename T>
void mySwap(T& a, T& b) {
    T temp = a;
    a = b;
    b = temp;
}

int main() {
    // 编译器自动推导类型
    std::cout << myMax(3, 7) << "\n";          // T = int，输出 7
    std::cout << myMax(3.14, 2.72) << "\n";    // T = double，输出 3.14
    std::cout << myMax('a', 'z') << "\n";      // T = char，输出 z

    // 显式指定模板参数
    std::cout << myMax<double>(3, 4.5) << "\n"; // 强制 T = double

    printPair(42, "hello");     // T=int, U=const char*
    printPair(3.14, true);      // T=double, U=bool

    int x = 10, y = 20;
    mySwap(x, y);
    std::cout << "x=" << x << ", y=" << y << "\n"; // x=20, y=10

    return 0;
}
💡

模板实例化：每次用不同类型调用模板函数时，编译器会生成一份对应的具体函数代码，这个过程称为实例化。myMax(3, 7) 和 myMax(3.14, 2.72) 会生成两份独立的函数。

📦 2. 类模板

类模板用于创建泛型容器和数据结构。标准库中的 std::vector<T>、std::stack<T> 等都是类模板。下面我们手动实现一个简单的 Stack<T>。

stack.hpp — 泛型栈 
#include <iostream>
#include <vector>
#include <stdexcept>

// 类模板：泛型栈
template<typename T>
class Stack {
private:
    std::vector<T> data_;   // 用 vector 存储元素

public:
    // 压栈
    void push(const T& val) {
        data_.push_back(val);
    }

    // 弹栈（返回并移除栈顶元素）
    T pop() {
        if (empty()) throw std::underflow_error("Stack is empty");
        T top = data_.back();
        data_.pop_back();
        return top;
    }

    // 查看栈顶（不移除）
    const T& top() const {
        if (empty()) throw std::underflow_error("Stack is empty");
        return data_.back();
    }

    bool empty() const { return data_.empty(); }
    size_t size() const { return data_.size(); }
};

int main() {
    Stack<int> intStack;
    intStack.push(1);
    intStack.push(2);
    intStack.push(3);
    std::cout << "栈顶: " << intStack.top() << "\n";   // 3
    std::cout << "弹出: " << intStack.pop() << "\n";   // 3
    std::cout << "大小: " << intStack.size() << "\n";  // 2

    Stack<std::string> strStack;
    strStack.push("hello");
    strStack.push("world");
    std::cout << strStack.pop() << "\n";  // world

    return 0;
}
📌

类模板成员函数的定义：如果在类外定义成员函数，需要重复写 template<typename T>，并用 Stack<T>:: 限定作用域。通常将类模板的实现直接放在 .hpp 头文件中，因为编译器需要看到完整定义才能实例化。

🔍 3. 模板参数推导

编译器通常能从函数调用的实参类型自动推导模板参数，无需手动指定。理解推导规则有助于避免常见错误。

参数推导规则示例 
#include <iostream>
#include <type_traits>

template<typename T>
void show_type(T val) {
    if constexpr (std::is_integral_v<T>)
        std::cout << val << " 是整数类型\n";
    else if constexpr (std::is_floating_point_v<T>)
        std::cout << val << " 是浮点类型\n";
    else
        std::cout << val << " 是其他类型\n";
}

// 引用参数推导：T 不包含引用修饰
template<typename T>
void ref_demo(T& val) {
    // T 推导为去掉引用后的类型
    std::cout << "sizeof(T) = " << sizeof(T) << "\n";
}

// 常见陷阱：混合类型无法自动推导
template<typename T>
T add(T a, T b) { return a + b; }

int main() {
    show_type(42);       // T = int
    show_type(3.14);     // T = double
    show_type('A');      // T = char

    // add(1, 2.0);   // 错误！1 推导 T=int，2.0 推导 T=double，冲突
    add(1, 2);         // OK: T = int
    add(1.0, 2.0);     // OK: T = double
    add<double>(1, 2.0); // OK: 显式指定 T=double，int 隐式转换

    // C++17 类模板参数推导（CTAD）
    // std::pair p{1, 3.14};   // 自动推导为 pair<int, double>
    // std::vector v{1, 2, 3}; // 自动推导为 vector<int>

    int x = 10;
    ref_demo(x);   // T = int（不是 int&）

    return 0;
}
⚠️

推导失败的常见场景：当实参类型不一致（如 add(1, 2.0)），或参数出现在非推导上下文（如返回类型、默认参数）时，编译器无法自动推导，需显式指定模板参数。

🔧 4. 模板特化基础

有时通用模板对某些特定类型需要不同的实现。全特化为特定类型提供专门的版本，编译器在实例化时优先选择特化版本。

全特化示例 
#include <iostream>
#include <cstring>
#include <string>

// 主模板：通用比较
template<typename T>
bool isEqual(T a, T b) {
    return a == b;
}

// 全特化：对 const char* 使用字符串比较
template<>
bool isEqual<const char*>(const char* a, const char* b) {
    return std::strcmp(a, b) == 0;
}

// 类模板全特化示例
template<typename T>
struct Printer {
    static void print(T val) {
        std::cout << val << "\n";
    }
};

// 对 bool 类型特化：输出 "true"/"false" 而不是 1/0
template<>
struct Printer<bool> {
    static void print(bool val) {
        std::cout << (val ? "true" : "false") << "\n";
    }
};

int main() {
    std::cout << isEqual(42, 42) << "\n";          // true（通用版本）
    std::cout << isEqual(3.14, 3.15) << "\n";      // false

    const char* s1 = "hello";
    const char* s2 = "hello";
    // 指针比较地址，结果不确定；特化版本比较内容
    std::cout << isEqual(s1, s2) << "\n";          // true（特化版本）

    Printer<int>::print(42);        // 42
    Printer<double>::print(3.14);   // 3.14
    Printer<bool>::print(true);     // true（特化版本）
    Printer<bool>::print(false);    // false

    return 0;
}
💡

全特化 vs 偏特化：全特化（template<>）指定了所有模板参数的具体类型；偏特化只指定部分参数或参数模式（如所有指针类型 T*），是进阶主题，将在模板元编程中深入介绍。

🔢 5. 非类型模板参数

模板参数不仅可以是类型，还可以是编译期常量值（整数、枚举、指针等）。标准库的 std::array<T, N> 就用非类型参数指定数组大小。

非类型模板参数示例 
#include <iostream>
#include <stdexcept>

// N 是编译期常量，作为数组大小
template<typename T, size_t N>
class FixedArray {
private:
    T data_[N];  // 栈上固定大小数组，无堆分配开销

public:
    FixedArray() = default;

    T& operator[](size_t idx) {
        if (idx >= N) throw std::out_of_range("Index out of range");
        return data_[idx];
    }

    const T& operator[](size_t idx) const {
        if (idx >= N) throw std::out_of_range("Index out of range");
        return data_[idx];
    }

    constexpr size_t size() const { return N; }

    void fill(const T& val) {
        for (size_t i = 0; i < N; ++i) data_[i] = val;
    }
};

// 编译期计算：阶乘模板
template<int N>
struct Factorial {
    static constexpr int value = N * Factorial<N - 1>::value;
};

template<>
struct Factorial<0> {
    static constexpr int value = 1;
};

// 非类型参数与类型参数混合
template<typename T, int Rows, int Cols>
class Matrix {
private:
    T data_[Rows][Cols]{};
public:
    T& at(int r, int c) { return data_[r][c]; }
    constexpr int rows() const { return Rows; }
    constexpr int cols() const { return Cols; }
};

int main() {
    FixedArray<int, 5> arr;
    arr.fill(0);
    arr[0] = 10; arr[2] = 30; arr[4] = 50;
    for (size_t i = 0; i < arr.size(); ++i)
        std::cout << arr[i] << " ";
    std::cout << "\n";  // 10 0 30 0 50

    // 不同 N 是不同类型
    FixedArray<int, 3> small;
    // small = arr;  // 错误！FixedArray<int,3> 和 FixedArray<int,5> 是不同类型

    // 编译期计算
    static_assert(Factorial<5>::value == 120);
    std::cout << "5! = " << Factorial<5>::value << "\n";  // 120
    std::cout << "8! = " << Factorial<8>::value << "\n";  // 40320

    // 3x3 矩阵
    Matrix<double, 3, 3> mat;
    mat.at(0, 0) = 1.0; mat.at(1, 1) = 2.0; mat.at(2, 2) = 3.0;
    std::cout << "对角元素: "
              << mat.at(0,0) << ", "
              << mat.at(1,1) << ", "
              << mat.at(2,2) << "\n";

    return 0;
}
📌

与 std::array 对比：std::array<T, N> 本质上就是对固定大小 C 数组的封装，N 是非类型模板参数。相比 std::vector，它无堆分配，大小在编译期确定，适合性能敏感场景。

🧪 综合示例：泛型排序与打印

templates_demo.cpp 
#include <iostream>
#include <algorithm>
#include <string>

// 泛型打印数组
template<typename T, size_t N>
void printArray(const T (&arr)[N], const char* label = "") {
    std::cout << label << "[";
    for (size_t i = 0; i < N; ++i) {
        std::cout << arr[i];
        if (i + 1 < N) std::cout << ", ";
    }
    std::cout << "]\n";
}

// 泛型冒泡排序（要求 T 支持 operator<）
template<typename T>
void bubbleSort(T* arr, size_t n) {
    for (size_t i = 0; i < n - 1; ++i)
        for (size_t j = 0; j < n - 1 - i; ++j)
            if (arr[j] > arr[j + 1])
                std::swap(arr[j], arr[j + 1]);
}

// 泛型查找：返回第一个满足条件的元素索引，-1 表示未找到
template<typename T, typename Pred>
int findIf(const T* arr, size_t n, Pred pred) {
    for (size_t i = 0; i < n; ++i)
        if (pred(arr[i])) return static_cast<int>(i);
    return -1;
}

int main() {
    int nums[] = {5, 2, 8, 1, 9, 3, 7};
    printArray(nums, "排序前: ");

    bubbleSort(nums, 7);
    printArray(nums, "排序后: ");

    // 查找第一个大于 5 的元素
    int idx = findIf(nums, 7, [](int x){ return x > 5; });
    if (idx >= 0)
        std::cout << "第一个 >5 的元素: " << nums[idx]
                  << "（索引 " << idx << "）\n";

    // 同样的模板用于字符串数组
    std::string words[] = {"banana", "apple", "cherry", "date"};
    printArray(words, "原始: ");
    bubbleSort(words, 4);
    printArray(words, "字典序: ");

    return 0;
}

📋 课后作业

理解题

解释为什么类模板的实现（成员函数定义）通常必须放在头文件中，而普通类可以将声明和实现分开放在 .h 和 .cpp 中。提示：与编译器的实例化时机有关。

编程题

实现一个类模板 Pair<T1, T2>，包含两个成员 first 和 second，支持构造函数初始化和 swap() 方法。再实现一个辅助函数模板 makePair(a, b)，让编译器自动推导类型并返回 Pair<T1, T2> 对象。

挑战题

实现一个函数模板 minElement 和 maxElement，接受 C 风格数组（使用非类型模板参数推导大小），返回数组中最小/最大值的引用。再为 const char* 类型的比较提供全特化版本，使其按字典序而非地址比较。用 int、double、std::string 三种类型验证。
