🔨 构造函数

构造函数（Constructor）是一种特殊的成员函数，与类同名，无返回值类型（连 void 都不写）。对象被创建时，构造函数自动调用，用于初始化对象的数据成员。

构造函数的各种形式 
#include <iostream>
#include <string>
using namespace std;

class Person {
    string name;
    int    age;
public:
    // 1. 默认构造函数（无参）
    Person() : name("未知"), age(0) {
        cout << "默认构造：" << name << endl;
    }

    // 2. 参数化构造函数
    Person(string n, int a) : name(n), age(a) {
        cout << "参数构造：" << name << endl;
    }

    // 3. 委托构造函数（C++11）：调用同类的另一个构造函数
    Person(string n) : Person(n, 0) {   // 委托给双参构造函数
        cout << "委托构造：" << name << endl;
    }

    // 4. 构造函数重载
    Person(int a) : name("匿名"), age(a) {
        cout << "年龄构造：age=" << age << endl;
    }

    void print() const {
        cout << name << "（" << age << "岁）" << endl;
    }
};

int main() {
    Person p1;              // 调用默认构造函数
    Person p2("Alice", 30); // 调用参数化构造函数
    Person p3("Bob");       // 调用委托构造函数
    Person p4(25);          // 调用年龄构造函数

    p2.print();  // Alice（30岁）
    p3.print();  // Bob（0岁）
    return 0;
}

📋 初始化列表（重要！）

初始化列表（Member Initializer List）是构造函数冒号 : 后面的部分，用于在进入函数体之前直接初始化成员变量。

初始化列表 vs 函数体赋值 
class Point {
    const int x;   // const 成员！
    int& ref;      // 引用成员！
    int y;

public:
    // ✓ 正确：const 和引用成员必须用初始化列表
    Point(int xVal, int& r, int yVal)
        : x(xVal), ref(r), y(yVal) {}

    // ✗ 错误写法（无法编译）：
    // Point(int xVal, int& r, int yVal) {
    //     x = xVal;   // 错误！const 成员不能赋值
    //     ref = r;    // 错误！引用不能重绑
    // }
};

class Expensive {
    string data;  // string 有自己的构造函数
public:
    // ✓ 高效：直接调用 string 的构造函数初始化
    Expensive(string s) : data(s) {}

    // ✗ 低效：先调用 string 默认构造，再赋值（两步操作）
    // Expensive(string s) { data = s; }
};
⚠️

初始化顺序：成员的初始化顺序取决于它们在类中的声明顺序，而不是初始化列表中的顺序！如果初始化列表顺序与声明顺序不同，某些编译器会给出警告。最佳实践：让初始化列表顺序与成员声明顺序保持一致。

为什么初始化列表更高效？

  • 对于内置类型（int, double 等）：性能相同
  • 对于类类型（string, vector 等）：初始化列表直接调用拷贝构造函数，而函数体赋值会先调用默认构造再赋值——多一次操作
  • 对于 const 成员和引用成员：必须用初始化列表，别无选择

⚙️ 默认构造函数与 = default

默认构造函数是无参（或所有参数都有默认值）的构造函数。编译器会在以下情况自动生成一个默认构造函数：没有用户自定义任何构造函数。

默认构造函数与 = default 
class A {
    int x;
    // 没有定义任何构造函数，编译器自动生成默认构造函数
};

class B {
    int x;
public:
    // 定义了带参构造函数，编译器不再自动生成默认构造
    B(int val) : x(val) {}
};

class C {
    int x;
public:
    C(int val) : x(val) {}

    // 显式要求编译器生成默认构造函数（C++11）
    C() = default;

    // 显式删除拷贝构造函数（禁止拷贝）
    C(const C&) = delete;
};

int main() {
    A a;      // 正确：使用编译器生成的默认构造
    // B b;   // 错误！B 没有默认构造函数
    C c(42);  // 正确：使用参数化构造
    C c2;     // 正确：使用 = default 显式要求的默认构造
    // C c3 = c2;  // 错误！拷贝构造被 delete
    return 0;
}
📌

= default 和 = delete 是 C++11 的特性。= default 告诉编译器"请生成你默认会生成的版本"（即使我定义了其他构造函数也要生成）；= delete 明确禁止某个函数，尝试调用会在编译期报错。

📋 拷贝构造函数

拷贝构造函数（Copy Constructor）在用一个同类对象初始化另一个新对象时被调用。

浅拷贝 vs 深拷贝 
#include <iostream>
#include <cstring>
using namespace std;

// 浅拷贝的问题：两个对象共享同一内存
class ShallowString {
    char* data;
public:
    ShallowString(const char* s) {
        data = new char[strlen(s) + 1];
        strcpy(data, s);
    }
    // 编译器默认生成的拷贝构造函数是浅拷贝
    // 只复制指针值，两个对象的 data 指向同一内存！
    ~ShallowString() { delete[] data; }  // double-free 崩溃！
};

// 深拷贝：为副本分配独立内存
class DeepString {
    char* data;
    int   len;
public:
    DeepString(const char* s) : len(strlen(s)) {
        data = new char[len + 1];
        strcpy(data, s);
    }

    // 深拷贝构造函数：独立复制内存
    DeepString(const DeepString& other) : len(other.len) {
        data = new char[len + 1];   // 分配新内存
        strcpy(data, other.data);    // 复制内容
        cout << "深拷贝构造：" << data << endl;
    }

    ~DeepString() { delete[] data; }  // 安全释放自己的内存
    const char* str() const { return data; }
};

int main() {
    DeepString s1("Hello");
    DeepString s2 = s1;    // 调用拷贝构造函数（深拷贝）
    DeepString s3(s1);     // 同上

    // 拷贝构造的触发时机：
    // 1. DeepString s2 = s1;     （用现有对象初始化）
    // 2. DeepString s3(s1);      （同上）
    // 3. void foo(DeepString s)  → foo(s1);   （按值传参）
    // 4. DeepString bar() { return DeepString("x"); }  （按值返回）

    return 0;
}
🚫

三/五法则（Rule of Three/Five）：如果你需要自定义析构函数（因为有动态内存），那么你几乎肯定也需要自定义拷贝构造函数和拷贝赋值运算符（C++11 还加上了移动构造和移动赋值）。不遵守这条规则是内存 bug 的主要来源。

💥 析构函数

析构函数（Destructor）在对象生命周期结束时自动调用，负责释放对象持有的资源。

析构函数与资源管理 
#include <iostream>
#include <cstdio>
using namespace std;

class FileManager {
    FILE* file;
    string filename;
public:
    FileManager(const char* path) : filename(path) {
        file = fopen(path, "r");
        if (file) cout << "打开文件：" << filename << endl;
    }

    // 析构函数：~ + 类名，无参数，无返回值
    ~FileManager() {
        if (file) {
            fclose(file);
            cout << "自动关闭文件：" << filename << endl;
        }
    }

    bool isOpen() const { return file != nullptr; }
};

// 析构顺序演示
class Base {
public:
    Base()  { cout << "Base 构造\n"; }
    ~Base() { cout << "Base 析构\n"; }
};
class Member {
public:
    Member()  { cout << "Member 构造\n"; }
    ~Member() { cout << "Member 析构\n"; }
};
class Derived : public Base {
    Member m;  // 成员对象
public:
    Derived()  { cout << "Derived 构造\n"; }
    ~Derived() { cout << "Derived 析构\n"; }
};

int main() {
    Derived d;
    // 构造顺序：Base → Member → Derived
    // 析构顺序：Derived → Member → Base（与构造相反）
    return 0;
}
💡

析构顺序规则：析构顺序严格与构造顺序相反。先构造的后析构。对于类的成员：先构造基类，再按声明顺序构造成员，最后执行构造函数体；析构时完全倒序。这保证了对象始终处于一致状态。

🛡️ RAII 原则

RAII（Resource Acquisition Is Initialization，资源获取即初始化）是 C++ 最重要的编程惯例之一。其核心思想是：

  • 资源（内存、文件、网络连接、锁等）在构造函数中获取
  • 资源在析构函数中释放
  • 对象的生命周期管理资源的生命周期
RAII 的威力 
#include <iostream>
#include <mutex>
using namespace std;

// ✗ 不使用 RAII 的危险代码
void dangerousFunction() {
    int* data = new int[1000];
    // ...做一些操作...
    if (someCondition) {
        return;   // 内存泄漏！忘记 delete
    }
    // ...更多操作...
    // 如果中途抛出异常，也会内存泄漏
    delete[] data;
}

// ✓ RAII 类：确保资源总是被释放
class ScopedArray {
    int* data;
public:
    ScopedArray(int size) : data(new int[size]) {
        cout << "分配内存\n";
    }
    ~ScopedArray() {
        delete[] data;
        cout << "自动释放内存\n";
    }
    int& operator[](int i) { return data[i]; }
};

// ✓ 使用 RAII 的安全代码
void safeFunction() {
    ScopedArray arr(1000);  // 构造时分配内存
    // ...无论如何返回（正常或异常），arr 的析构函数都会被调用
    if (someCondition) {
        return;  // arr 自动释放，绝无内存泄漏
    }
}   // arr 超出作用域，析构函数自动调用

对象生命周期时序图：

RAII：构造获取资源 → 存活期使用 → 析构自动释放

📌

RAII 的好处：无论函数如何退出（正常返回、提前 return、还是抛出异常），资源都会被正确释放。这是 C++ 程序比 C 程序更安全的核心原因。标准库的 std::string、std::vector、std::unique_ptr 都是 RAII 的典范。
