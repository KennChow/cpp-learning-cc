💡 OOP 的核心思想

面向对象编程（Object-Oriented Programming，OOP）是一种将数据（属性）与操作数据的函数（方法）封装在一起的编程范式。现实世界中，一辆汽车有颜色、速度（数据），也有加速、刹车（操作），OOP 就是把这两者绑定成一个"对象"。

与结构体（struct）相比，class 默认成员是 private（结构体默认 public），这让数据封装更加安全自然。C++ 支持四大 OOP 特性：

🔒
封装 Encapsulation
将数据和方法绑定，隐藏内部实现细节，只暴露必要的接口
🧬
继承 Inheritance
子类复用父类的代码，体现"is-a"关系（第3课详讲）
🔀
多态 Polymorphism
同一接口，不同实现；运行时动态决定调用哪个版本（第4课详讲）
📐
抽象 Abstraction
用抽象类定义接口，隐藏不必要的细节，只展示关键行为
💡

struct vs class：C++ 中 struct 和 class 几乎相同，唯一区别是默认访问权限。struct 默认 public，class 默认 private。惯例上，简单的数据聚合用 struct，有封装行为的对象用 class。

🏗️ class 的定义与使用

下面是一个银行账户类的完整示例，展示了 class 的核心组成部分：

BankAccount 类定义 
#include <iostream>
#include <string>
using namespace std;

class BankAccount {
private:
    string owner;       // 私有数据成员：外部无法直接访问
    double balance;

public:
    // 构造函数（对象创建时自动调用）
    BankAccount(string name, double initialBalance)
        : owner(name), balance(initialBalance) {}  // 初始化列表

    // 存款
    void deposit(double amount) {
        if (amount > 0) balance += amount;
    }

    // 取款（返回是否成功）
    bool withdraw(double amount) {
        if (amount > balance) return false;
        balance -= amount;
        return true;
    }

    // const 成员函数：承诺不修改对象状态
    double getBalance() const { return balance; }
    string getOwner()  const { return owner; }

    void printInfo() const {
        cout << owner << " 的账户余额：" << balance << " 元" << endl;
    }
};

int main() {
    // 在栈上创建对象
    BankAccount acc("张三", 1000.0);
    acc.deposit(500.0);
    acc.withdraw(200.0);
    acc.printInfo();      // 张三 的账户余额：1300 元

    // 在堆上创建对象
    BankAccount* pAcc = new BankAccount("李四", 2000.0);
    pAcc->deposit(300.0);
    pAcc->printInfo();    // 李四 的账户余额：2300 元
    delete pAcc;          // 必须手动释放

    return 0;
}

🔐 访问控制详解

访问控制是封装的核心机制，C++ 提供三种访问说明符：

访问说明符 可访问范围 典型用途
public 任何地方（类内、类外、派生类） 对外接口（成员函数、构造函数）
protected 类自身 + 派生类 供子类使用但不对外公开的成员
private 仅类自身的成员函数 内部数据、辅助函数（默认）

为什么要用 private 而不是直接公开变量？

  • 数据验证：通过 setter 函数可以在赋值前检查合法性（如存款金额不能为负）
  • 维护不变量：余额不能小于 0，直接访问 balance 可能破坏这个约束
  • 可维护性：内部实现改变（如换存储格式）不影响外部调用者
getter / setter 与 const 成员函数 
class Temperature {
private:
    double celsius;   // 存储摄氏度

public:
    // setter：在赋值时进行验证
    void setTemp(double c) {
        if (c < -273.15) {
            throw std::invalid_argument("温度不能低于绝对零度");
        }
        celsius = c;
    }

    // const getter：不修改对象，可在 const 对象上调用
    double getCelsius()    const { return celsius; }
    double getFahrenheit() const { return celsius * 9.0/5.0 + 32; }
    double getKelvin()     const { return celsius + 273.15; }
};

// const 对象只能调用 const 成员函数
const Temperature freezing{};  // celsius = 0
// freezing.setTemp(100);  // 错误！不能修改 const 对象
double f = freezing.getFahrenheit();  // 正确：const 成员函数
⚠️

const 成员函数的承诺：函数签名末尾的 const 表示该函数承诺不会修改对象的任何数据成员。这让 const 对象（和 const 引用）也能调用这些函数。养成习惯：所有不修改状态的 getter 都应标记为 const。

👉 this 指针

this 是一个隐式指针，在每个非静态成员函数内部自动可用，它指向当前正在调用该函数的对象。

this 指针与链式调用 
#include <iostream>
using namespace std;

class Counter {
    int count = 0;
public:
    // 返回 *this 引用，实现链式调用
    Counter& increment() {
        count++;
        return *this;    // 返回自身的引用
    }

    Counter& add(int n) {
        count += n;
        return *this;
    }

    Counter& reset() {
        count = 0;
        return *this;
    }

    void print() const {
        cout << "count = " << count << endl;
    }
};

int main() {
    Counter c;
    // 链式调用：每次都返回同一个对象的引用
    c.increment().increment().add(5).increment().print();
    // count = 8

    // 解析链式调用：
    // c.increment()  → count=1, 返回 c 的引用
    // .increment()   → count=2, 返回 c 的引用
    // .add(5)        → count=7, 返回 c 的引用
    // .increment()   → count=8, 返回 c 的引用
    // .print()       → 输出 count = 8

    return 0;
}

this 指针的其他用途：

  • 解决命名冲突：当参数名与成员名相同时，用 this->name 区分
  • 传递自身：将当前对象的指针传给其他函数
  • 链式调用：返回 *this 实现 Builder 模式等设计模式
this 解决命名冲突 
class Person {
    string name;
    int age;
public:
    // 参数名与成员名相同，用 this-> 区分
    void setInfo(string name, int age) {
        this->name = name;   // this->name 是成员，name 是参数
        this->age  = age;
    }
};

🏭 对象实例化

C++ 中有多种方式创建对象，各有适用场景：

对象创建方式对比 
#include <iostream>
#include <vector>
using namespace std;

class Point {
public:
    double x, y;
    Point(double x = 0, double y = 0) : x(x), y(y) {}
    void print() const { cout << "(" << x << ", " << y << ")" << endl; }
};

int main() {
    // 1. 栈上创建（自动管理生命周期，函数结束自动销毁）
    Point p1(3.0, 4.0);
    p1.print();               // (3, 4)

    // 2. 堆上创建（手动管理，用完必须 delete）
    Point* p2 = new Point(1.0, 2.0);
    p2->print();              // (1, 2)
    delete p2;                // 释放内存

    // 3. 对象数组（栈上）
    Point points[3] = { {0,0}, {1,1}, {2,2} };
    for (const auto& p : points) p.print();

    // 4. 动态对象数组（堆上）
    Point* arr = new Point[5];   // 调用默认构造函数 5 次
    delete[] arr;                 // 注意用 delete[]，不是 delete

    // 5. 推荐：使用智能指针（C++11，自动释放）
    // unique_ptr<Point> p3 = make_unique<Point>(5.0, 6.0);

    return 0;
}
💡

优先使用栈上对象：栈上对象自动管理生命周期，速度更快，不会内存泄漏。只有在需要动态大小、跨函数传递、或者对象很大时才考虑堆分配。现代 C++ 推荐用 unique_ptr / shared_ptr 管理堆上对象（进阶课讲解）。
