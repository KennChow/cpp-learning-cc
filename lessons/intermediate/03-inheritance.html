🧬 什么是继承？

继承（Inheritance）是一种代码复用机制，允许一个类（派生类/子类）获得另一个类（基类/父类）的成员。继承体现的是"is-a"关系：

  • Dog（狗）is-a Animal（动物）✓
  • Car（汽车）is-a Vehicle（交通工具）✓
  • Stack（栈）is-a Array（数组）✗ —— 这是"has-a"关系，不应该用继承

继承的优势：子类自动拥有父类的数据和方法，只需在子类中定义新增或修改的部分，避免重复代码。

Animal → Dog / Cat 继承示例 
#include <iostream>
#include <string>
using namespace std;

// 基类（父类）
class Animal {
protected:                  // protected：子类可访问，外部不行
    string name;
    int    age;
public:
    Animal(string n, int a) : name(n), age(a) {}

    void eat() { cout << name << " 在吃东西\n"; }

    virtual void speak() {  // virtual：允许子类重写（第4课详讲）
        cout << name << " 发出声音...\n";
    }

    void printInfo() const {
        cout << "名字: " << name << ", 年龄: " << age << "\n";
    }
};

// 派生类（子类）— public 继承
class Dog : public Animal {
    string breed;  // Dog 自己的数据成员
public:
    // 子类构造函数：通过初始化列表调用父类构造函数
    Dog(string n, int a, string b)
        : Animal(n, a), breed(b) {}

    // 重写父类的虚函数
    void speak() override {
        cout << name << " 汪汪叫！\n";  // 可访问 protected 成员 name
    }

    // Dog 特有的方法
    void fetch() { cout << name << " 去取球了\n"; }
};

class Cat : public Animal {
public:
    Cat(string n, int a) : Animal(n, a) {}

    void speak() override {
        cout << name << " 喵喵叫！\n";
    }

    void purr() { cout << name << " 呼噜呼噜...\n"; }
};

int main() {
    Dog dog("小白", 3, "柴犬");
    Cat cat("咪咪", 2);

    // 继承来的方法
    dog.eat();       // 小白 在吃东西
    dog.printInfo(); // 名字: 小白, 年龄: 3

    // 子类特有方法
    dog.speak();   // 小白 汪汪叫！（覆盖了父类的 speak）
    dog.fetch();   // 小白 去取球了
    cat.speak();   // 咪咪 喵喵叫！

    return 0;
}

🔑 三种继承方式

C++ 有三种继承方式，决定了基类成员在派生类中的可见性：

基类成员访问 public 继承 protected 继承 private 继承
public 成员 public protected private
protected 成员 protected protected private
private 成员 不可访问 不可访问 不可访问
💡

实践建议：绝大多数情况下使用 public 继承，它保留了"is-a"语义。protected 和 private 继承较少使用，它们更像是"根据...实现"（implemented-in-terms-of），语义上更接近组合（has-a）。

⏱️ 构造与析构的调用顺序

继承体系中，构造和析构的顺序有明确的规则：

构造析构顺序演示 
#include <iostream>
using namespace std;

class A {
public:
    A()  { cout << "A 构造\n"; }
    ~A() { cout << "A 析构\n"; }
};

class B : public A {
    int x;
public:
    B(int v) : A(), x(v) {    // 先调用 A()，再初始化 x
        cout << "B 构造\n";
    }
    ~B() { cout << "B 析构\n"; }  // B 的析构完成后再调用 A 的析构
};

class C : public B {
public:
    C() : B(42) {  // 先调用 B(42)（B 内部又调用 A()）
        cout << "C 构造\n";
    }
    ~C() { cout << "C 析构\n"; }
};

int main() {
    C obj;
    // 构造顺序（从根类到派生类）：
    // A 构造
    // B 构造
    // C 构造

    // 析构顺序（构造的反顺序）：
    // C 析构
    // B 析构
    // A 析构
    return 0;
}

// 关键规则：
// 构造：先父类（根 → 叶），后成员，最后本类函数体
// 析构：先本类函数体，后成员，最后父类（叶 → 根，与构造相反）
⚠️

子类构造函数必须调用父类构造函数：如果父类没有默认构造函数，子类构造函数必须在初始化列表中显式调用父类构造函数。否则编译器会尝试调用父类的默认构造函数，如果不存在则编译报错。

🔒 override 和 final（C++11）

override 和 final 是 C++11 引入的关键字（上下文关键字），帮助程序员更安全地处理虚函数。

override 与 final 的用法 
class Base {
public:
    virtual void foo(int x) { /* ... */ }
    virtual void bar() { /* ... */ }
};

class Derived : public Base {
public:
    // ✓ override：告诉编译器"我打算覆盖父类的虚函数"
    // 如果签名不匹配，编译器会报错（防止拼写错误）
    void foo(int x) override { /* ... */ }

    // ✗ 没有 override 时，拼写错误会静默创建新函数，而非覆盖
    // void fooo(int x) { }   // 悄悄创建了新函数，不是 bug

    // ✓ final：禁止子类继续覆盖这个虚函数
    void bar() override final { /* ... */ }
};

class Further : public Derived {
    // void bar() override { }  // 错误！bar() 被标记为 final，不能覆盖
};

// final 也可以用于类：禁止继续派生
class Sealed final : public Base {
    void foo(int x) override { /* ... */ }
};
// class TryExtend : public Sealed { };  // 错误！Sealed 是 final 类
📌

养成好习惯：覆盖虚函数时始终加上 override。这能在编译期捕获签名不匹配的错误，比运行时调用错误函数要好得多。大型代码库中，缺少 override 是重构时的重要 bug 来源。

🔀 多重继承与菱形问题

C++ 支持多重继承（一个类同时继承多个基类），这是 C++ 区别于 Java/C# 的特性之一。

多重继承与菱形问题 
// 多重继承的基本语法
class Flyable {
public:
    void fly() { cout << "飞行\n"; }
};

class Swimmable {
public:
    void swim() { cout << "游泳\n"; }
};

// Duck 同时继承两个接口
class Duck : public Flyable, public Swimmable {
public:
    void quack() { cout << "嘎嘎！\n"; }
};

Duck d;
d.fly();   // 来自 Flyable
d.swim();  // 来自 Swimmable
d.quack(); // Duck 自己的方法

// ============ 菱形继承问题 ============
class Animal { int id; };
class Dog : public Animal { };
class Cat : public Animal { };
// DogCat 会有两份 Animal 的数据成员！
class DogCat : public Dog, public Cat { };
// DogCat dc;
// dc.id;  // 错误！歧义：是 Dog::Animal::id 还是 Cat::Animal::id？

// 解决方案：虚继承
class VAnimal { int id; };
class VDog : virtual public VAnimal { };  // 虚继承
class VCat : virtual public VAnimal { };
class VDogCat : public VDog, public VCat { };  // 只有一份 VAnimal 数据
VDogCat vdc;
// vdc.id;  // 正确！只有一份 id
⚠️

多重继承要谨慎：多重继承增加了复杂性（菱形问题、名字冲突）。现代 C++ 推荐用纯虚接口（类似 Java interface）进行多重继承，而不是多重实现继承。如果你觉得需要多重继承，先考虑是否能用组合（has-a）替代。

📋 课后作业

理解题

解释"is-a"和"has-a"关系的区别，并为以下场景判断应该用继承还是组合：(1) 学生和人；(2) 汽车和引擎；(3) 正方形和矩形（注意里氏替换原则）；(4) 经理和员工。

编程题

创建 Shape 基类（包含 name 和 color 字符串成员），派生 Circle（半径）和 Rectangle（长宽），每个子类都有构造函数（通过初始化列表调用父类构造）以及 area() 和 describe() 方法。用 override 标注覆盖的函数。

挑战题
