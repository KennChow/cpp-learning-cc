📝 你的第一个 C++ 程序

下面是一个最简单的 C++ 程序，它在屏幕上打印一行文字：

hello.cpp 
// 引入输入输出流头文件
#include <iostream>

// 程序从这里开始执行
int main() {
    // 向标准输出（控制台）打印文字
    std::cout << "Hello, World!" << std::endl;

    // 返回 0 表示程序正常结束
    return 0;
}

逐行解析：

代码 含义
#include <iostream> 引入标准输入输出流库，使程序可以使用 cout 和 cin
int main() 程序的入口函数，C++ 程序必须从 main 开始执行，返回类型 int
{ ... } 大括号定义函数体（代码块），所有语句必须写在其中
std::cout << "..." << std::endl; 向控制台输出字符串，<< 是流插入运算符，endl 换行并刷新缓冲区
return 0; 向操作系统返回退出码，0 表示程序正常结束，非 0 表示出错

⚙️ 编译流程

C++ 是编译型语言。你写的 .cpp 源文件不能直接运行，需要经过一系列工具处理才能变成操作系统可执行的文件：

C++ 完整编译流程：源码 → 预处理 → 编译 → 汇编 → 链接 → 可执行文件

💡

实际使用时，命令 g++ hello.cpp -o hello 会自动完成上述所有步骤。选项 -o hello 指定输出文件名。

🔍

可以用 g++ -E hello.cpp 只运行预处理阶段；用 g++ -S hello.cpp 生成汇编代码；用 g++ -c hello.cpp 只生成目标文件。

📦 #include 与头文件

#include 是一个预处理指令（preprocessor directive），它告诉预处理器将指定文件的内容复制插入到当前文件中。

include 的两种形式 
// 尖括号：搜索标准库头文件目录
#include <iostream>    // 标准输入输出
#include <string>      // std::string
#include <vector>      // std::vector

// 双引号：先搜索当前目录，再搜索系统目录
#include "myheader.h"  // 你自己写的头文件

<iostream> 提供了以下常用功能：

  • std::cout — 标准输出流（输出到控制台）
  • std::cin — 标准输入流（从键盘读取）
  • std::cerr — 标准错误流（输出错误信息）
  • std::endl — 换行符并刷新输出缓冲区

🗂️ 命名空间与 using namespace std

命名空间（namespace）是一种将相关名称（函数、类、变量）分组在一起的机制，防止不同库之间发生命名冲突。

三种使用 std 的方式 
#include <iostream>

// 方式 1：完全限定名（推荐，清晰明确）
int main() {
    std::cout << "方式1" << std::endl;
}

// 方式 2：using 声明（只引入特定名称）
using std::cout;
using std::endl;
int main() {
    cout << "方式2" << endl;
}

// 方式 3：using namespace（引入全部，不推荐在头文件中使用）
using namespace std;
int main() {
    cout << "方式3" << endl;
}
⚠️

不要在头文件中写 using namespace std;，这会"污染"所有包含该头文件的文件的命名空间，可能造成意外的命名冲突。在 .cpp 文件中使用相对安全，但在大型项目中仍建议使用限定名 std::。

🖨️ std::cout 与输出

std::cout 是一个输出流对象，<< 是流插入运算符，可以链式调用：

cout 的各种用法 
#include <iostream>

int main() {
    // 输出字符串
    std::cout << "Hello, World!" << std::endl;

    // 输出数字
    std::cout << 42 << std::endl;
    std::cout << 3.14 << std::endl;

    // 链式输出（一行多项）
    int age = 20;
    std::cout << "我今年 " << age << " 岁" << std::endl;

    // \n vs endl：\n 更快，endl 会额外刷新缓冲区
    std::cout << "第一行\n";
    std::cout << "第二行" << std::endl;

    return 0;
}
🚀

性能提示：std::endl 等同于 '\n' + flush。如果不需要立即刷新缓冲区，用 '\n' 或 "\n" 代替 std::endl 速度更快，在循环中输出大量数据时差异明显。

💬 注释

注释是给程序员看的说明文字，编译器会忽略注释。良好的注释习惯是专业编程的重要素养。

注释的两种形式 
// 这是单行注释，从 // 到行尾都是注释

/* 这是多行注释
   可以跨越多行
   用 /* 开始，*/ 结束 */

int main() {
    // 单行注释常用于简短说明
    int x = 10;  // 也可以在代码后面

    /* 多行注释常用于：
       1. 临时注释掉一段代码
       2. 函数或类的文档说明 */

    return 0;
}
📌

现代 C++ 项目广泛使用 Doxygen 风格注释（/** ... */ 或 ///）来自动生成 API 文档。注释应解释"为什么"这样做，而不是"做了什么"（代码本身已经说明了做什么）。

🔙 return 0 的含义

main 函数的返回值是程序的退出状态码，被操作系统用来判断程序是否成功执行：

返回值 含义 Shell 中检查方式
0 程序正常结束（成功） echo $? → 0
1 (非零) 程序异常结束（失败） echo $? → 1
EXIT_SUCCESS / EXIT_FAILURE 标准库定义的常量（可移植性更好） <cstdlib>
ℹ️

C++11 起，如果 main 函数没有 return 语句，编译器会自动在末尾插入 return 0;。但为了代码清晰，建议总是显式写出 return 0;。
