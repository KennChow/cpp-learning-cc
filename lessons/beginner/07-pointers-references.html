指针与引用

内存地址 指针 解引用 引用 动态内存 重要

🎯 学习目标

  • ✓ 理解内存地址的概念，掌握 & 取地址运算符
  • ✓ 掌握指针的声明、初始化、解引用和指针算术
  • ✓ 理解空指针 nullptr 及其重要性
  • ✓ 掌握引用的使用，理解其与指针的本质区别

🏠 内存和地址

程序运行时，每个变量都存储在内存中的某个位置。每个位置都有一个唯一的地址（十六进制数字）。& 运算符可以获取变量的地址：

取地址运算符 
int x = 42;
double d = 3.14;
char c = 'A';

cout << "x 的值: " << x << endl;
cout << "x 的地址: " << &x << endl;  // 类似 0x7fff5b3c4a1c
cout << "d 的地址: " << &d << endl;
cout << "c 的地址: " << (void*)&c << endl;

// 地址的大小
// 32 位系统：地址占 4 字节（0x00000000 ~ 0xFFFFFFFF）
// 64 位系统：地址占 8 字节（现代计算机常见）
cout << "地址大小: " << sizeof(&x) << " 字节" << endl;  // 通常 8
📌

地址的本质：内存是一个巨大的字节数组，地址就是数组下标。int x 在内存中占 4 个字节（地址 N、N+1、N+2、N+3），&x 返回的是起始地址 N。

👆 指针基础

指针是一种变量，它的值是另一个变量的内存地址。通过指针，你可以间接访问和修改它所指向的变量：

指针的声明、初始化与解引用 
int x = 42;

// 声明指针：类型* 变量名（推荐写法：* 紧靠类型）
int* p;           // p 是"指向 int 的指针"（未初始化，危险！）
int* p = &x;     // p 存储 x 的地址（推荐：声明时立即初始化）

// 解引用：* 运算符读取指针指向的值
cout << *p << endl;   // 42（读取 x 的值）
*p = 100;              // 通过指针修改 x 的值
cout << x << endl;    // 100（x 被修改了！）

// 指针的指针
int** pp = &p;         // pp 存储 p 的地址
cout << **pp << endl; // 100（解引用两次）

// 两种星号写法，效果相同，推荐前者
int* p1;   // 推荐：强调 p1 是"int*类型"
int *p2;   // 传统：强调 *p2 是"int类型"

图解：指针变量指向另一个变量

指针变量 p 存储的是 x 的内存地址。通过 *p（解引用）可以读取或修改 x 的值。

🚫 空指针 nullptr

未初始化的指针（野指针）是非常危险的，它指向随机地址。当指针不指向任何有效对象时，应该将其设为 nullptr（C++11 引入，替代旧式 NULL）：

nullptr 的正确使用 
// 空指针：不指向任何有效对象
int* p = nullptr;  // 推荐：明确表示"无效指针"

// 使用前必须检查是否为空
if (p != nullptr) {
    *p = 42;  // 安全
}
// 或者简写：
if (p) { *p = 42; }

// 常见错误：解引用空指针 → 段错误（Segmentation Fault）
int* q = nullptr;
// *q = 5;  // 崩溃！未定义行为

// C++11 之前用 NULL（本质是整数 0，不推荐）
int* old = NULL;  // 不推荐

// nullptr 是类型安全的，NULL 不是
// nullptr 只能赋值给指针类型，NULL 可能被误用为整数

图解：空指针与解引用危险

🔢 指针算术

指针可以进行加减运算。p + 1 不是地址加 1，而是移动一个元素大小的距离。这是指针与数组密切相关的原因：

指针算术与数组 
int arr[4] = {10, 20, 30, 40};

int* p = arr;  // 数组名 = 指向首元素的指针

// 指针+整数：向后移动 n 个元素
cout << *p << endl;       // 10（arr[0]）
cout << *(p + 1) << endl; // 20（arr[1]）
cout << *(p + 3) << endl; // 40（arr[3]）

// arr[i] 完全等价于 *(arr + i)
// arr[2] == *(arr + 2) == *(p + 2) == 30

// 指针递增
p++;           // p 现在指向 arr[1]
cout << *p << endl;  // 20

// 两指针相减 = 元素个数差
int* begin = arr;
int* end = arr + 4;
cout << end - begin << endl;  // 4

// 注意：对 int* 加1，实际地址增加 sizeof(int) = 4 字节
cout << (void*)arr << endl;       // 例如 0x1000
cout << (void*)(arr+1) << endl;   // 例如 0x1004（+4字节）

图解：指针算术与数组

🔗 引用

引用（reference）是变量的别名——它不是新变量，而是已有变量的另一个名字。对引用的所有操作都直接作用于原变量：

引用的使用 
int x = 10;
int& ref = x;   // ref 是 x 的引用（别名）

cout << ref << endl;  // 10（和 x 一样）
ref = 99;              // 修改 ref 就是修改 x
cout << x << endl;    // 99

// 引用的特点：
// 1. 必须在声明时初始化
// int& r;    // 错误！引用必须初始化

// 2. 不能重新绑定到另一个变量
// ref = y;   // 这是赋值（把 y 的值赋给 x），不是重新绑定

// 3. 常用于函数参数：避免拷贝，允许修改
void increment(int& n) {
    n++;  // 直接修改调用者的变量
}
increment(x);  // x 变为 100

// 4. const 引用：只读不改（常用于大型对象参数）
void print(const string& s) {
    cout << s << endl;  // 不能修改 s
}
💡

引用 vs 指针的使用场景：函数参数用引用更简洁（无需 * 解引用，不能为 null）；当参数可能为"无值"状态时用指针；动态内存管理必须用指针。const T& 是传递大型只读参数的最佳方式。

⚖️ 指针 vs 引用对比

特性 指针（int*） 引用（int&）
可以为空 ✓ nullptr ✗ 不存在空引用
可以重新指向 ✓ p = &y ✗ 绑定后不变
需要解引用 ✓ *p ✗ 直接用名字
有自己的地址 ✓ &p 有意义 通常是被引用变量的地址
初始化要求 可以先声明后赋值 必须声明时初始化
典型用途 动态内存、可选参数 函数参数、别名

🌐 动态内存分配

栈上的局部变量在函数返回后自动销毁，大小必须在编译期已知。堆（Heap）允许在运行时动态分配任意大小的内存，但需要手动管理：

new / delete 动态内存 
// ===== 单个对象 =====
int* p = new int;        // 在堆上分配一个 int
*p = 42;                 // 通过指针赋值
cout << *p << endl;     // 42
delete p;                // 释放内存（必须！）
p = nullptr;             // 立即置空，避免悬空指针

// 分配时初始化
double* d = new double(3.14);  // 分配并初始化为 3.14
delete d;
d = nullptr;

// ===== 数组 =====
int n = 5;  // 运行时决定大小
int* arr = new int[n];   // 分配 n 个 int 的数组
for (int i = 0; i < n; i++) arr[i] = i * i;
for (int i = 0; i < n; i++) cout << arr[i] << " ";
delete[] arr;   // 数组必须用 delete[]（不能用 delete）
arr = nullptr;

// ===== 常见错误 =====
// 内存泄漏：忘记 delete
// int* q = new int[100];  // 函数返回后，内存无法回收

// 双重释放（UB）
// delete p; delete p;  // 第二次 delete 是未定义行为

// 悬空指针（dangling pointer）
// delete p; cout << *p;  // p 仍指向已释放的内存，UB

图解：栈与堆的对比

⚠️ 常见陷阱总结

1. 野指针（Wild Pointer）

未初始化的指针指向随机内存，解引用会导致不可预知的错误。解决：声明时立即初始化为 nullptr 或有效地址。

2. 悬空指针（Dangling Pointer）

delete 后忘记置 nullptr，指针仍指向已释放的内存。解决：delete 后立即 p = nullptr。

3. 内存泄漏（Memory Leak）

用 new 分配的内存忘记 delete，程序长时间运行会消耗越来越多内存。解决：每个 new 对应一个 delete；或使用智能指针。

4. 双重释放（Double Free）

同一指针 delete 两次，破坏堆管理器状态。解决：delete 后立即置 nullptr，因为 delete nullptr 是安全的无操作。

📌

现代 C++ 的解决方案：上述问题可以通过智能指针（std::unique_ptr、std::shared_ptr）自动解决——它们会在作用域结束时自动释放内存。智能指针将在中级课程中详细介绍。

🧪 综合代码示例

指针与引用综合演示 
#include <iostream>
using namespace std;

void swap_by_pointer(int* a, int* b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

void swap_by_reference(int& a, int& b) {
    int temp = a;
    a = b;
    b = temp;
}

int main() {
    // 基本指针
    int x = 42;
    int* p = &x;
    cout << "x 的值: " << x << endl;
    cout << "x 的地址: " << &x << endl;
    cout << "p 指向: " << p << endl;
    cout << "*p (解引用): " << *p << endl;

    *p = 100;  // 通过指针修改 x
    cout << "修改后 x = " << x << endl;

    // 指针算术
    int arr[4] = {10, 20, 30, 40};
    int* q = arr;
    cout << "q[0]=" << *q << " q[1]=" << *(q+1) << endl;

    // 引用交换
    int a = 5, b = 10;
    swap_by_reference(a, b);
    cout << "交换后: a=" << a << " b=" << b << endl;

    // 动态内存
    int* heap_arr = new int[5];
    for (int i = 0; i < 5; i++) heap_arr[i] = i * i;
    for (int i = 0; i < 5; i++) cout << heap_arr[i] << " ";
    cout << endl;
    delete[] heap_arr;
    heap_arr = nullptr;

    return 0;
}
