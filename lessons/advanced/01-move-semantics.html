移动语义与右值引用

C++11 移动语义 完美转发 性能优化

🎯 学习目标

  • ✓ 理解值类别：左值（lvalue）、右值（rvalue）、将亡值（xvalue）的区别
  • ✓ 掌握右值引用 && 的语法和使用场景
  • ✓ 能够编写移动构造函数和移动赋值运算符
  • ✓ 理解 std::move 的本质（类型转换，不移动数据）
  • ✓ 掌握 std::forward 实现完美转发

📌 1. 值类别：左值与右值

C++ 中每个表达式都有一个值类别（value category），决定了它能否绑定到特定类型的引用。理解值类别是掌握移动语义的基础。

左值（lvalue）

有名字、有持久地址的表达式。可以出现在赋值符号左侧。

int x = 42; // x 是左值
std::string s = "hello"; // s 是左值

右值（rvalue）

临时对象、字面量，没有持久地址。通常只出现在赋值右侧。

42 // 纯右值（prvalue）
x + y // 纯右值（prvalue）
std::move(s) // 将亡值（xvalue）

C++ 值类别分类树（C++11 起）

🔗 2. 右值引用 &&

C++11 引入了右值引用，用 T&& 表示。右值引用只能绑定到右值（临时对象），这是移动语义的语法基础。

右值引用基础 
int x = 10;

// 左值引用：只能绑定到左值
int& lref = x;      // OK
// int& lref2 = 42; // 错误！不能绑定到右值

// 右值引用：只能绑定到右值
int&& rref = 42;    // OK，绑定到字面量（右值）
int&& rref2 = x + 1; // OK，x+1 是临时值（右值）
// int&& rref3 = x; // 错误！x 是左值

// const 左值引用：可以绑定到右值（但不能修改）
const int& cref = 42; // OK，特殊规则

std::cout << rref << std::endl;  // 输出 42

注意：右值引用变量本身是左值

一旦右值引用被命名（如 int&& r = 42;），r 本身就成为左值，因为它有名字和地址。这是完美转发需要 std::forward 的根本原因。

🚀 3. 移动构造函数与移动赋值

移动语义让我们能够将资源从一个对象"转移"到另一个对象，而不是进行昂贵的深拷贝。对于持有动态内存的类，移动操作可以将 O(n) 的拷贝降为 O(1) 的指针转移。

移动构造函数实现 
class MyString {
    char* data_;
    size_t size_;
public:
    // 普通构造
    MyString(const char* str) {
        size_ = strlen(str);
        data_ = new char[size_ + 1];
        memcpy(data_, str, size_ + 1);
        std::cout << "构造: " << data_ << "\n";
    }

    // 拷贝构造（深拷贝，O(n)）
    MyString(const MyString& other) {
        size_ = other.size_;
        data_ = new char[size_ + 1];
        memcpy(data_, other.data_, size_ + 1);
        std::cout << "拷贝构造: " << data_ << "\n";
    }

    // 移动构造（资源转移，O(1)）
    MyString(MyString&& other) noexcept  // 注意 noexcept！
        : data_(other.data_), size_(other.size_) {
        other.data_ = nullptr;  // 源对象失去资源所有权
        other.size_ = 0;
        std::cout << "移动构造\n";
    }

    // 移动赋值运算符
    MyString& operator=(MyString&& other) noexcept {
        if (this != &other) {
            delete[] data_;          // 释放自身旧资源
            data_ = other.data_;     // 接管资源
            size_ = other.size_;
            other.data_ = nullptr;   // 源对象置空
            other.size_ = 0;
        }
        return *this;
    }

    ~MyString() {
        delete[] data_;
        std::cout << "析构\n";
    }

    const char* c_str() const { return data_ ? data_ : ""; }
};

为什么移动构造函数应该标记 noexcept？

  • std::vector 等容器在重新分配内存时，只有当移动构造函数是 noexcept 时才会使用移动，否则退回到拷贝（保证强异常安全）
  • 移动操作通常不应该失败（只是指针转移），标记 noexcept 是正确的语义表达

⚡ 4. std::move — 转型而非移动

std::move 的名字有些误导性。它本身不移动任何数据，只是将一个表达式无条件转型为右值引用，告诉编译器"这个对象的资源可以被转移走"。

std::move 的使用 
// std::move 的简化实现（实际是 static_cast）
template<typename T>
typename std::remove_reference<T>::type&& move(T&& t) noexcept {
    return static_cast<typename std::remove_reference<T>::type&&>(t);
}

// 使用示例
MyString s1("Hello, World!");  // 构造

MyString s2 = s1;              // 拷贝构造（s1 仍然有效）
MyString s3 = std::move(s1);   // 移动构造（s1 的资源被转移，s1 变为空）

std::cout << s3.c_str() << "\n";  // "Hello, World!"
// s1 现在处于"有效但未指定"状态，不应再使用

// vector 中的移动
std::vector<MyString> vec;
vec.push_back(MyString("temp"));  // 临时对象被移动进去，而不是拷贝

使用 std::move 后的注意事项

被 std::move 后的对象处于"有效但未指定（valid but unspecified）"状态。你可以对它赋新值或销毁它，但不应该读取其内容或依赖其值。

📊 5. 移动 vs 拷贝：性能对比

对于管理动态内存的类（如字符串、容器），移动操作比拷贝快得多：

拷贝（深拷贝）O(n)

  1. 分配新内存块
  2. 逐字节复制数据
  3. 复制 size 等元数据

1MB 的字符串需要复制 100 万字节

移动（指针转移）O(1)

  1. 复制指针（8 字节）
  2. 复制 size 等元数据
  3. 将源指针置为 nullptr

无论数据多大，始终只操作几个字节

vector 扩容时的移动优化 
#include <vector>
#include <string>

// std::vector 扩容时：
// - 如果元素类型有 noexcept 移动构造，使用移动（快）
// - 否则使用拷贝（慢，但保证异常安全）

std::vector<std::string> v;
v.reserve(3);

v.push_back("one");    // 可能移动临时 string
v.push_back("two");
v.push_back("three");

// 当 vector 需要扩容时，已有元素会被移动（而非拷贝）到新内存块
// std::string 有 noexcept 移动构造，所以 vector 会使用移动

🎯 6. 完美转发 std::forward

在模板函数中，当我们需要将参数原封不动地转发给另一个函数时，需要保持参数的值类别（左值仍为左值，右值仍为右值）。这就是完美转发。

完美转发示例 
#include <utility>

void process(MyString& s)  { std::cout << "左值版本\n"; }
void process(MyString&& s) { std::cout << "右值版本\n"; }

// 不使用完美转发（有问题）
template<typename T>
void wrapper_bad(T&& t) {
    // t 是有名字的变量，所以始终是左值
    // 无论传入左值还是右值，都会调用左值版本
    process(t);  // 总是调用 process(MyString&)
}

// 使用完美转发（正确）
template<typename T>
void wrapper_good(T&& t) {
    // std::forward<T> 在 T 是左值引用时返回左值，否则返回右值
    process(std::forward<T>(t));  // 保持值类别
}

// make_unique, emplace_back 等都使用了完美转发
template<typename T, typename... Args>
std::unique_ptr<T> my_make_unique(Args&&... args) {
    return std::unique_ptr<T>(new T(std::forward<Args>(args)...));
}

int main() {
    MyString s("hello");

    wrapper_good(s);              // 传入左值 → 调用左值版本
    wrapper_good(MyString("hi")); // 传入右值 → 调用右值版本
    wrapper_good(std::move(s));   // 转为右值 → 调用右值版本
}

🧪 综合代码示例

完整的移动语义演示，展示拷贝 vs 移动的行为差异：

move_semantics_demo.cpp 
#include <iostream>
#include <utility>
#include <vector>
#include <cstring>

class Buffer {
    int* data_;
    size_t size_;
public:
    explicit Buffer(size_t n) : size_(n), data_(new int[n]) {
        std::fill(data_, data_ + n, 0);
        std::cout << "[构造] size=" << n << " @ " << data_ << "\n";
    }

    // 拷贝构造
    Buffer(const Buffer& o) : size_(o.size_), data_(new int[o.size_]) {
        std::copy(o.data_, o.data_ + o.size_, data_);
        std::cout << "[拷贝] " << o.data_ << " → " << data_ << "\n";
    }

    // 移动构造
    Buffer(Buffer&& o) noexcept : size_(o.size_), data_(o.data_) {
        o.data_ = nullptr;
        o.size_ = 0;
        std::cout << "[移动] 接管 " << data_ << "\n";
    }

    // 移动赋值
    Buffer& operator=(Buffer&& o) noexcept {
        if (this != &o) {
            delete[] data_;
            data_ = o.data_;
            size_ = o.size_;
            o.data_ = nullptr;
            o.size_ = 0;
            std::cout << "[移动赋值]\n";
        }
        return *this;
    }

    ~Buffer() {
        if (data_) {
            std::cout << "[析构] @ " << data_ << "\n";
            delete[] data_;
        } else {
            std::cout << "[析构] (已被移走，无需释放)\n";
        }
    }

    size_t size() const { return size_; }
    bool valid() const { return data_ != nullptr; }
};

// 工厂函数：返回临时对象（RVO/NRVO 优化可能消除拷贝）
Buffer make_buffer(size_t n) {
    return Buffer(n);
}

int main() {
    std::cout << "=== 拷贝语义 ===\n";
    Buffer b1(1024);
    Buffer b2 = b1;  // 深拷贝：两份数据
    std::cout << "b1.valid=" << b1.valid() << " b2.valid=" << b2.valid() << "\n\n";

    std::cout << "=== 移动语义 ===\n";
    Buffer b3(512);
    Buffer b4 = std::move(b3);  // 移动：b3 失去所有权
    std::cout << "b3.valid=" << b3.valid() << " b4.valid=" << b4.valid() << "\n\n";

    std::cout >> "=== 工厂函数（RVO）===\n";
    Buffer b5 = make_buffer(256);  // 通常 RVO 优化，无拷贝/移动
    std::cout << "b5.size=" << b5.size() << "\n\n";

    std::cout << "=== vector 中的移动 ===\n";
    std::vector<Buffer> vec;
    vec.reserve(2);
    vec.push_back(Buffer(100));   // 临时对象被移动进去
    vec.push_back(std::move(b5)); // b5 被移动进去
    std::cout << "vec.size=" << vec.size() << "\n\n";

    std::cout << "=== 程序结束，析构所有对象 ===\n";
    return 0;
}

/* 输出示例：
[构造] size=1024 @ 0x...
[拷贝] 0x... → 0x...
b1.valid=1 b2.valid=1

[构造] size=512 @ 0x...
[移动] 接管 0x...
b3.valid=0 b4.valid=1
...
*/
